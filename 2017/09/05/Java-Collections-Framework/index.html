Java Collections Framework

发表于 Sep 05, 2017

Java Collections Framework 是 Java SE 提供的容器的接口定义与实现，包括：

  • Interfaces. 抽象的容器接口定义，如 Set、Map
  • Implementations. 具体的容器实现，如 HashSet、TreeMap
  • Algorithms. 对容器对象的计算和操作，如排序、查找，支持多态

Java SE 官方文档

Interfaces

Iterable

Iterable 不算是 Java Collections Framework 中的一员，它是定义在 java.lang 包中的一个基础的接口，所有实现该接口的类都可以通过迭代器 Iterator 来遍历其中的对象。

在 Java 语言规范中，仅有实现 Iterable 接口或者数组类型的对象能以 enhanced for statement （for-each）的方式遍历，否则会在编译时报错。

Iterable 中定义了方法 iterator(), 返回可以迭代该对象的 Iterator；

在 Java 1.8 中，Iterable 又加入了一个 default 方法 forEach() 来支持函数式编程的 for-each 循环，以及一个 default 方法 spliterator()，返回可以并行迭代对象的迭代器 Spliterator，这里不多做介绍了。

Collection

Collection 是除 Map 族以外所有容器的基类，仅表示存放一组对象的容器。在 Collection 的定义中，元素可以是重复的，也可以是不重复的；可以是排序的，也可以是不排序的；取决于具体的子类或实现。注意它与数学中的集合概念的区别。

Collection 中定义了一些所有容器通用的基本操作：

JDK 中没有提供直接实现 Collection 接口的类，而是在其基础上又继承出一些子接口，如：Set、List、Queue 等。而在这些子接口中，我们没有看到描述一组无序且允许重复的对象的集合 —— MultiSet 。

在 JDK Bug System (JBS) 列表中，JDK-6567419 提到了这点，评论有说是 MultiSet、MultiMap 使用场景不是很多，The Collections Framework 的作者们没什么热情做。。。

抛开 JDK 本身， Guava 和 Apache Commons 这两个著名的 Collections 第三方扩展包中均实现了这个数据结构。其接口直接继承了 Collection，与 Set、List 等接口在同一层级。这个在 JDK 的 Collection 接口说明中也有提到：

Bags or multisets (unordered collections that may contain duplicate elements) should implement this interface directly.

List

List 在中文里一般叫做列表，其描述一组有序的元素，直接继承了 Collection 接口。与 Collection 接口相比，List 并没有其他的新增方法定义，也仅仅只是重新定义了一遍。

Set

Set 与数学中的集合概念更贴近，其描述一组不可重复的元素，直接继承了 Collection 接口。
