用 PhpStorm 和 SFTP 搭建高效的开发环境

发表于 Nov 02, 2016

使用PhpStorm的好处

  • 容易发现语法错误（Vim不易发现）
  • 智能补全，效率++
  • 一键Rename重构，不需要再grep
  • 可集成各种工具（sonar、phpcs等）
  • 断网也能干活

本地搭建环境的问题

  • 开发用的电脑是Macbook，公司的php扩展（tair，mcc，rabbitmq）安装不怎么支持，环境搭建困难
  • 不容易模拟线上机器环境
  • 本机还要安装Nginx、php-fpm，配置用户等，也挺麻烦

IDE != 本地搭建

通常我们的观念都是用IDE就等于需要本地开发调试，然而……

PhpStorm的强大之处在于：可以搭建SFTP部署服务器，实时同步本地与远程机器的文件（加强版自动rsync！）

当然首先我们得在远程机器搭建好SFTP服务（不能是FTP）……

首先搭好SSH服务

问题：公司限制本机无法直接ssh到自己的办公云机器

解决：我们有办公云root权限啊！

然后本机就可以通过2222端口ssh到远程办公云了！

搭建FTP服务

PhpStorm设置远程部署

路径：PhpStorm → Preferences → Build, Execution, Deployment → Deployment

新建，Type选择SFTP

Connection里就是些连接配置

Mappings设置本地目录与server目录的映射（即要把哪个目录的文件同步到办公云的哪个目录）

开始High

比较low的模式：每次更改完，想同步的时候，右键项目→ Deployment → Upload To SFTP server

轻松模式：Tools→ Deployment → 勾选Automatic Upload

这样每次command+s保存文件的时候就可以自动同步到办公云了！

IDE支持Phalcon

IDE不支持Phalcon框架，老是报一堆类、函数未找到也很困扰呀……

解决：https://github.com/phalcon/phalcon-devtools

下载下来，然后在PhpStorm中项目的External Libraries里面右键Configure PHP Include Paths，加上phalcon-devtools的/ide/stubs/Phalcon目录引入即可。
